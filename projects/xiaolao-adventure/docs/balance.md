## 1. 核心属性系统 (StatsSystem)

### 1.1 基础属性

| 属性 | 缩写 | 说明 | 初始值 | 成长系数 |
|------|------|------|--------|----------|
| 生命值 | HP | Health Points | 100 | +20/级 |
| 攻击力 | ATK | Attack | 10 | +3/级 |
| 防御力 | DEF | Defense | 5 | +2/级 |
| 速度 | SPD | Speed | 10 | +2/级 |

### 1.2 派生属性

- **最大生命值**: `baseHP + (level - 1) * 20`
- **战斗攻击力**: `baseATK + weaponATK + buffATK`
- **战斗防御力**: `baseDEF + armorDEF + buffDEF`
- **行动顺序**: 按 SPD 排序，相同则随机

## 2. 战斗系统 (BattleSystem)

### 2.1 回合制流程

```
1. 计算行动顺序（SPD）
2. 每个单位回合：
   - 选择行动（攻击/技能/物品/逃跑）
   - 执行行动
   - 检查战斗结束条件
3. 回合结束，进入下一回合
```

### 2.2 伤害公式

```
基础伤害 = max(1, 攻击者.ATK - 防御者.DEF * 0.5)
暴击伤害 = 基础伤害 * 1.5  (暴击率 = 5% + SPD * 0.2%)
最终伤害 = 基础伤害 * random(0.9, 1.1)  // ±10% 浮动
```

**设计决策**: 
- 防御只抵消 50% 攻击力，避免高防无敌
- 速度影响暴击率，让速度有价值
- 随机浮动增加不确定性

### 2.3 经验值系统

```
击败获得经验 = 敌人等级 * 10 + 敌人HP * 0.1
升级所需经验 = 100 * level^1.5
```

**设计决策**: 1.5次方曲线让后期升级变慢，保持挑战性

## 4. 经济系统 (EconomySystem)

### 4.1 货币

- **金币 (G)** - 主要货币
- **钻石 (💎)** - 高级货币（充值/稀有掉落）

### 4.2 收入来源

| 来源 | 金额 | 说明 |
|------|------|------|
| 击败敌人 | 等级*5~15G | 随机浮动 |
| 任务奖励 | 100~500G | 根据难度 |
| 出售物品 | 原价*0.6 | 折旧 |

### 4.3 商店系统

```
购买价格 = 基础价格
出售价格 = 基础价格 * 0.6
```

---

## 5. 数值平衡原则

1. **战斗时长**: 普通战斗 3-5 回合，Boss 战 8-15 回合
2. **药水价值**: 一瓶药水 ≈ 30% 最大 HP
3. **升级感**: 每升一级属性提升 10-15%，有明显成长感
4. **经济循环**: 战斗收入 ≈ 药水消耗 + 装备折旧
