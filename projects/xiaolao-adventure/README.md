# 虾游记 (Xiaolao Adventure)

> 作者：@虾老大

虾游记是一个使用 TypeScript 构建的文字冒险游戏项目，由三只虾协作开发：一只负责游戏引擎，一只负责剧情体验，一只负责战斗与数值设计。

项目当前聚焦于构建一个**模块化、可测试、可扩展**的游戏核心：玩家在剧情分支中推进故事，在关键节点进入回合制战斗，并通过状态管理与存档系统保存自己的冒险进度。

---

## 游戏简介

**虾游记（Xiaolao Adventure）** 是一款以文字叙事为核心的冒险游戏。

玩家将跟随“小劳”的旅程，在海滩、村庄、洞窟等场景中探索世界，通过对话选项影响剧情走向，并在遭遇敌人时进入回合制战斗。整个项目强调“剧情 + 战斗 + 引擎架构”三者协同，是一个既能玩、也适合持续开发演进的游戏工程。

### 玩法特色

- **分支剧情体验**：玩家可以通过选项推进剧情，触发不同对话与状态变化。
- **回合制战斗系统**：支持攻击、使用物品、逃跑等经典回合制行为。
- **模块化引擎设计**：剧情、战斗、场景、状态、存档分离，方便维护和扩展。
- **可持久化进度**：支持通过状态与存档系统记录游戏进程。
- **适合协作开发**：架构清晰，便于不同模块并行演进。

## 安装部署

### 环境要求

在开始之前，请确认你的环境满足以下条件：

- `Node.js` 18 或更高版本
- `npm` 9 或更高版本
- 推荐使用 macOS、Linux 或 Windows + WSL

### 安装步骤

1. 克隆项目仓库

```bash
git clone <your-repo-url>
cd xiaolao-adventure
```

2. 安装依赖

```bash
npm install
```

3. 编译 TypeScript 源码

```bash
npm run build
```

4. 运行项目

```bash
npm start
```

### 开发模式

如果你希望直接运行 TypeScript 源码进行开发调试：

```bash
npm run dev
```

### 测试命令

运行全部测试：

```bash
npm test
```

查看覆盖率：

```bash
npm run test:coverage
```

### 常见问题

#### 1. 依赖安装失败怎么办？

可以尝试：

```bash
npm cache clean --force
npm install
```

#### 2. `npm start` 没有看到完整游戏界面？

当前项目重点在于核心引擎、剧情和战斗逻辑层，运行结果更偏向逻辑执行与模块验证。随着后续 UI 接入，交互体验会更完整。

## 游戏玩法

### 如何开始游戏

当前项目的核心玩法由引擎模块驱动，基础流程如下：

1. 启动游戏引擎
2. 进入起始场景
3. 开始剧情对话
4. 通过选项推进剧情
5. 在特定节点触发战斗
6. 保存或恢复当前进度

### 基本操作概念

虽然当前仓库更偏向逻辑层实现，但玩法逻辑已经比较清晰：

- **开始游戏**：由 `GameEngine.start()` 初始化场景与剧情。
- **推进剧情**：通过 `GameEngine.progressStory(optionId)` 选择对话选项。
- **进入战斗**：通过 `GameEngine.startBattle(player, enemies)` 触发战斗。
- **推进回合**：通过战斗系统执行动作，再由主循环同步状态。
- **切换场景**：由 `SceneManager.transitionTo(sceneId)` 记录场景流转。
- **保存进度**：由 `SaveManager.save()` 序列化当前状态。
- **读取进度**：由 `SaveManager.load()` 恢复场景与状态。

### 一局游戏的大致流程

```text
开始游戏
  ↓
进入初始场景（海滩）
  ↓
触发剧情对话
  ↓
玩家选择分支
  ↓
更新状态 / 切换场景
  ↓
触发战斗（如有）
  ↓
结算结果
  ↓
继续剧情或保存游戏
```

## 项目架构

虾游记当前采用三大核心模块协作的架构设计：

### 1. `engine/` 游戏引擎模块

由 **虾老大** 负责，主要解决“游戏如何运行”的问题。

包括：

- `GameEngine`：主循环与系统协调
- `SceneManager`：场景切换与历史记录
- `StateManager`：运行状态与持久化
- `SaveManager`：存档与读档

### 2. `story/` 剧情模块

由 **虾可爱** 负责，主要解决“故事如何展开”的问题。

包括：

- `StoryManager`：剧情统一入口
- `DialogueSystem`：对话节点、选项、分支推进
- 角色与场景数据：角色注册、剧情节点组织

### 3. `battle/` 战斗模块

由 **虾算盘** 负责，主要解决“战斗如何计算”的问题。

包括：

- `BattleSystem`：回合制战斗核心
- `StatsSystem`：角色属性与成长
- `ItemSystem`：物品与装备系统
- `EconomySystem`：经济和奖励计算

### 架构特点

- **引擎协调，领域解耦**
- **剧情与战斗独立发展**
- **状态与存档分层明确**
- **方便测试与扩展**

详细说明可查看：`docs/architecture.md:1`

## 开发团队

虾游记由三只虾分工协作完成：

- 🦐 **虾老大**：负责游戏引擎、场景管理、状态管理、存档系统与整体架构
- 🦐💕 **虾可爱**：负责剧情系统、角色塑造、对话分支与故事体验
- 🧮🦐 **虾算盘**：负责战斗系统、数值设计、属性成长与经济系统

### 协作方式

这三个角色对应三个主要开发方向：

- **引擎层**：定义运行机制和系统协作方式
- **内容层**：负责故事、人物、场景和文本体验
- **规则层**：负责战斗、成长、装备、收益与平衡

这种分工方式让每个模块都能独立迭代，同时又能通过统一接口集成到一起。

## 技术栈

虾游记当前使用以下技术栈：

### 核心语言与运行环境

- `TypeScript`：提供静态类型、接口定义和更可靠的重构体验
- `Node.js`：作为当前开发与执行环境

### 测试与工程化

- `Jest`：单元测试与覆盖率统计
- `ts-jest`：让 Jest 能直接运行 TypeScript 测试文件
- `ts-node`：用于开发模式直接执行 TypeScript 源码

### 当前工程特点

- 强类型模块边界
- 以单元测试保证稳定性
- 以文档辅助多人协作
- 适合逐步演进为更完整的游戏项目
